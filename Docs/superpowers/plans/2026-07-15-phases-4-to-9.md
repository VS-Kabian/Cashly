# Phases 4–9 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make Cashly verifiable, maintainable, scalable for a user’s transaction history, and more resilient and accessible without changing the financial data model.

**Architecture:** Keep Supabase as the system of record. Centralize React Query defaults and transaction list reads in a typed hook, paginate transaction history at the database boundary, and invalidate only affected cached queries after writes. Add pure-function tests and an operator-run database validation guide because this checkout is not linked to a live Supabase project.

**Tech Stack:** React 18, TypeScript, TanStack Query v5, Supabase JS v2, Node built-in test runner, Vite.

## Global Constraints

- Do not add a service-role key or any secret to the browser bundle.
- Do not modify or apply historical migrations; add an additive migration only when a schema change is necessary.
- Preserve existing URLs, user workflows, and the RLS ownership model from Phases 1–3.
- Write a failing focused test before each new pure behavior, then run the full test suite and production build.
- Agents have exclusive file scopes; do not edit another task’s files.
